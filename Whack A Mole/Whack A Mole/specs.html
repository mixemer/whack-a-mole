Software Requirements Specification

for Whack-a-Professor by Group 1 CISC 3140

1 Introduction

1.1 Purpose

The purpose of this page is to provide an overall description and understanding of the CISC 3140 project, Whack-a-Prof. This page will list all the specifications of the project to include font and font size, the game features, baseline requirements, and graphics.

1.2 Product Scope

The Whack-a-Prof software is a game designed in Javascript for the CISC 3140 course project.

1.3 Definitions, Acronyms and Abbreviations

In this document, User and Player refers to the end-user and intended audience.

2 Overall Description

2.1 Product Overview

Whack-a-Prof is a standalone software that is a variation on the traditional Whack-a-Mole game. The game is intended to run on any web browser that supports Javascript. The game is in a 3 by 3 grid arrangement where the Professor will pop up on the screens in which the user will then have to click on the Professor to “whack” him.

The Player has only 20 seconds to get as high of a score as possible with the mallet which the Player controls with the mouse.
The Professor pops up randomly and the score counter increases by 10 points when the Professor is hit.

2.2 Product Functionality

  • Simple Whack-a-Mole style game with timer
  • Professors pop up at random intervals
  • User must use mouse cursor to “hit” the Professors
  • User will gain points per successful hit.
  • Game must have sound effects to let User know whether the hit was successful or not.

2.3 Design and Implementation Constraints

The Whack-a-Prof should be designed to run in any web browser, to include mobile browsers. The game has to be written in vanilla Javascript and CSS.
Images must be in SVG format to maintain resolution independence and transparency. Audio files must be in either MP3 or WAV format.

3 Specific Requirements

3.1 External Interface Requirements

3.1.1 User Interfaces

Start Screen: The game can only begin when User presses the “Play” button. Upon clicking “Play”, the game begins and the timer starts counting down. The User must use the mouse or touchscreen to press the Professor.

Game Over Screen: The game is over when timer reaches 0 at which the timer will stop and no Professor will appear on the screen. The music will also stop playing when the timer reaches 0. The final score is displayed and the Player will have the chance to play again.

3.1.2 Hardware Requirements

The game is intended to run in a web browser and should be able to run on most devices that support modern browsers to include desktops, laptops, tablets, and mobile devices. In order to play the game, the user must have at least a mouse or touch screen, where applicable.
